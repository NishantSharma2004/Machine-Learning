Import Libraries & Load Dataset
pandas · numpy · matplotlib · seaborn
▾

The very first thing — import your tools and load your data into a DataFrame. Think of this as opening your toolbox and reading your raw material.

pandas numpy matplotlib seaborn
python
# ── STEP 1: Import Libraries & Load Data ──────────────
import pandas as pd

2
Exploratory Data Analysis (EDA)
Understand shape · distributions · correlations
▾

Before touching anything, understand your data. Look at statistics, distributions, and relationships between columns. This step tells you what problems to fix next.

What to check

  • Number of rows & columns
  • Data types of each column
  • Missing values count
  • Min, max, mean, std
  • Unique values per column

Visualizations

  • Histogram — value distribution
  • Heatmap — correlations
  • Boxplot — outliers
  • Scatter — relationships
  • Count plot — categories
python
# ── STEP 2: Exploratory Data Analysis ─────────────────
print(df.info())           # dtypes + null counts
print(df.describe())       # statistics for number columns
print(df.isnull().sum())   # missing values per column
print(df.duplicated().sum()) # duplicate rows

# Visualise correlations
plt.figure(figsize=(10, 8))
sns.heatmap(df.corr(), annot=True, cmap='coolwarm')
plt.title('Feature Correlation')
plt.show()

# Distribution of target column
sns.histplot(df['target_column'], kde=True)
plt.show()

# Outlier check with boxplot
sns.boxplot(x=df['any_numeric_column'])
plt.show()
3
Data Cleaning & Preprocessing
Handle nulls · remove duplicates · fix types
▾

Raw data is always messy. Fix missing values, remove duplicates, fix wrong data types, and handle outliers. Clean data = better model.

Missing Values

  • Fill with mean/median (numbers)
  • Fill with mode (categories)
  • Drop rows if very few missing
  • Drop column if >50% missing

Outliers

  • IQR method (remove extremes)
  • Z-score (remove > 3 std)
  • Cap values (clip)
  • Log transform to reduce skew
python
# ── STEP 3: Data Cleaning ──────────────────────────────
# Drop duplicates
df.drop_duplicates(inplace=True)

# Fill missing values
df['numeric_col'].fillna(df['numeric_col'].median(), inplace=True)
df['category_col'].fillna(df['category_col'].mode()[0], inplace=True)

# Drop column if too many nulls
df.drop(columns=['bad_column'], inplace=True)

# Fix data types
df['date_col'] = pd.to_datetime(df['date_col'])
df['num_col']  = df['num_col'].astype(float)

# Remove outliers using IQR
Q1  = df['price'].quantile(0.25)
Q3  = df['price'].quantile(0.75)
IQR = Q3 - Q1
df  = df[(df['price'] >= Q1 - 1.5*IQR) & (df['price'] <= Q3 + 1.5*IQR)]
4
Feature Engineering & Encoding
Encode categories · scale numbers · create features
▾

ML models only understand numbers. Convert text categories to numbers, scale numeric features so no column dominates, and create new useful features from existing ones.

Encoding Types

  • Label Encoding — 2 categories
  • One-Hot — multiple categories
  • Ordinal — ordered categories
  • Target encoding — high cardinality

Scaling Types

  • StandardScaler — mean=0, std=1
  • MinMaxScaler — range 0 to 1
  • RobustScaler — outlier-resistant
  • Log transform — skewed data
python
# ── STEP 4: Feature Engineering & Encoding ────────────
from sklearn.preprocessing import LabelEncoder, StandardScaler

# Label Encode binary text columns (Yes/No, Male/Female)
le = LabelEncoder()
df['gender'] = le.fit_transform(df['gender'])

# One-Hot Encode multi-category columns
df = pd.get_dummies(df, columns=['city', 'product_type'])

# Create new features (Feature Engineering)
df['age_squared']       = df['age'] ** 2
df['price_per_sqft']    = df['price'] / df['area']
df['log_income']        = np.log1p(df['income'])

# Separate Features (X) and Target (y)
X = df.drop(columns=['target_column'])
y = df['target_column']

# Scale features
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)
💡 Rule of thumb: Use StandardScaler for models like Linear Regression, SVM, KNN. Tree-based models (Random Forest, XGBoost) usually don't need scaling.
5
Split Data — Train & Test
80% train · 20% test · stratify for classification
▾

Split your data so the model learns on one part and is tested on unseen data. This mimics real-world usage and gives you an honest performance score.

python
# ── STEP 5: Train-Test Split ───────────────────────────
from sklearn.model_selection import train_test_split

X_train, X_test, y_train, y_test = train_test_split(
    X_scaled, y,
    test_size=0.2,      # 20% for testing
    random_state=42,   # reproducibility
    stratify=y          # use for classification only
)

print(f"Train: {X_train.shape}, Test: {X_test.shape}")

# Optional: Validation set (for tuning)
X_train, X_val, y_train, y_val = train_test_split(
    X_train, y_train, test_size=0.1, random_state=42
)
⚠️ Remove stratify=y for regression problems (house price, sales). Keep it for classification (spam/not spam, disease/no disease).

Choose & Train the Model
Regression · Classification · Ensemble
▾

Pick the right algorithm based on your problem type. Try multiple models and compare — there's no single best model for every dataset.

Regression (predict a number)

  • Linear Regression — baseline
  • Decision Tree Regressor
  • Random Forest Regressor
  • XGBoost / Gradient Boost

Classification (predict a class)

  • Logistic Regression — baseline
  • Decision Tree Classifier
  • Random Forest Classifier
  • SVM / KNN / XGBoost
python
# ── STEP 6: Choose & Train Model ──────────────────────
from sklearn.linear_model    import LinearRegression, LogisticRegression
from sklearn.tree            import DecisionTreeClassifier
from sklearn.ensemble        import RandomForestClassifier, RandomForestRegressor
from sklearn.neighbors       import KNeighborsClassifier

# ── For REGRESSION (house price, sales) ───
model = RandomForestRegressor(n_estimators=100, random_state=42)

# ── For CLASSIFICATION (polluted/clean, fraud/not) ───
# model = RandomForestClassifier(n_estimators=100, random_state=42)

# Train the model
model.fit(X_train, y_train)
print("✅ Model trained!")
7
Evaluate the Model
Accuracy · R² · MAE · Confusion Matrix
▾

Check how well your model actually performs on data it has never seen. Use different metrics depending on whether you're doing classification or regression.

Regression Metrics

  • MAE — average error in units
  • MSE — penalizes large errors
  • RMSE — error in original units
  • R² Score — % variance explained

Classification Metrics

  • Accuracy — % correct predictions
  • Precision — of predicted +ve, how many right
  • Recall — of actual +ve, how many found
  • F1-Score — balance of P & R
python
# ── STEP 7: Evaluate Model ────────────────────────────
from sklearn.metrics import (
    mean_absolute_error, mean_squared_error, r2_score,   # regression
    accuracy_score, classification_report, confusion_matrix  # classification
)

y_pred = model.predict(X_test)

# ── REGRESSION evaluation ─────────────────
mae  = mean_absolute_error(y_test, y_pred)
rmse = np.sqrt(mean_squared_error(y_test, y_pred))
r2   = r2_score(y_test, y_pred)
print(f"MAE : {mae:.2f}")
print(f"RMSE: {rmse:.2f}")
print(f"R²  : {r2:.4f}")  # closer to 1 = better

# ── CLASSIFICATION evaluation ──────────────
# print(accuracy_score(y_test, y_pred))
# print(classification_report(y_test, y_pred))
# sns.heatmap(confusion_matrix(y_test, y_pred), annot=True)

# Feature importance (tree models)
importances = pd.Series(model.feature_importances_, index=X.columns)
importances.sort_values().plot(kind='barh')
plt.title('Feature Importance')

8
Predict on New Data & Save Model
Real predictions · pickle · joblib
▾

Use your trained model to make real predictions. Save it to disk so you don't have to retrain every time — load and predict instantly.

python
# ── STEP 8: Predict & Save Model ──────────────────────
import joblib

# Save model to disk
joblib.dump(model,  'my_model.pkl')
joblib.dump(scaler, 'my_scaler.pkl')
print("💾 Model saved!")

# ── Later: Load and predict ───────────────
model  = joblib.load('my_model.pkl')
scaler = joblib.load('my_scaler.pkl')

# Predict on brand new single input
new_data = pd.DataFrame({
    'area'   : [1200],
    'rooms'  : [3],
    'age'    : [5],
    'location': [1]   # after encoding
})
new_scaled  = scaler.transform(new_data)
prediction  = model.predict(new_scaled)
print(f"Predicted Price: ₹{prediction[0]:,.0f}")
🚀 Bonus tip: Always save your scaler alongside your model. When predicting, new data must be transformed with the same scaler used during training — otherwise predictions will be completely wrong.
